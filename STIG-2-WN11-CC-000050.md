## What is the STIG ID?
**WN11-CC-000050** — Hardened UNC Paths must be defined to require mutual authentication and integrity for at least the `\\*\SYSVOL` and `\\*\NETLOGON` shares.

UNC paths like `\\domain\SYSVOL` and `\\domain\NETLOGON` deliver Group Policy and login scripts. Without hardening, an attacker could spoof these shares and inject malicious content. Hardened UNC Paths force the client to require mutual auth + integrity before trusting them.

## How I'm fixing it
Two REG_SZ values under one registry key. The key path doesn't exist by default, so we create it first.

| Field | Value |
|---|---|
| Key  | `HKLM\SOFTWARE\Policies\Microsoft\Windows\NetworkProvider\HardenedPaths` |
| Name | `\\*\SYSVOL` |
| Type | `REG_SZ` |
| Data | `RequireMutualAuthentication=1, RequireIntegrity=1` |

| Field | Value |
|---|---|
| Name | `\\*\NETLOGON` |
| Type | `REG_SZ` |
| Data | `RequireMutualAuthentication=1, RequireIntegrity=1` |

**PowerShell (run as Administrator):**

```powershell
$path = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\NetworkProvider\HardenedPaths"

# Create the key path if it doesn't exist
if (-not (Test-Path $path)) {
    New-Item -Path $path -Force | Out-Null
}

# Add both required UNC entries
New-ItemProperty -Path $path -Name "\\*\SYSVOL"   -Value "RequireMutualAuthentication=1, RequireIntegrity=1" -PropertyType String -Force
New-ItemProperty -Path $path -Name "\\*\NETLOGON" -Value "RequireMutualAuthentication=1, RequireIntegrity=1" -PropertyType String -Force

# Local verify
Get-ItemProperty -Path $path | Select-Object "\\*\SYSVOL", "\\*\NETLOGON"
```
