## What is the STIG ID?
**WN11-CC-000066** — Command line data must be included in process creation events.

By default, Windows logs Event ID 4688 (process creation) but only records the executable name. Enabling this policy adds the **full command line** to the event, which is critical for incident response — it's how analysts catch encoded PowerShell payloads, LOLBin abuse, and command arguments used by malware.

> **Note:** This STIG pairs with `WN11-AU-000050` (audit Detailed Tracking → Process Creation successes). This setting only adds command-line detail to events that are already being recorded; the audit policy is what makes them recorded in the first place.

## How I'm fixing it
Single registry DWORD. The key path may need to be created.

| Field | Value |
|---|---|
| Key  | `HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\Audit` |
| Name | `ProcessCreationIncludeCmdLine_Enabled` |
| Type | `REG_DWORD` |
| Data | `1` |

**PowerShell (run as Administrator):**

```powershell
$path = "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\Audit"

# Create key path if it doesn't exist
if (-not (Test-Path $path)) {
    New-Item -Path $path -Force | Out-Null
}

# Set the policy value
New-ItemProperty -Path $path -Name "ProcessCreationIncludeCmdLine_Enabled" -Value 1 -PropertyType DWORD -Force

# Local verify
Get-ItemProperty -Path $path -Name "ProcessCreationIncludeCmdLine_Enabled"
```
