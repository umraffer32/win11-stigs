# STIG #6 — WN11-SO-000190

<img width="1509" height="456" alt="image" src="https://github.com/user-attachments/assets/c30c7527-d915-450f-821f-963ecc154bd9" />

## What is the STIG ID?
**WN11-SO-000190** — Kerberos encryption types must be configured to prevent the use of DES and RC4 encryption suites.

Kerberos supports multiple encryption types ranging from legacy (DES, RC4) to modern (AES). DES is brute-forceable, and RC4-HMAC enables **Kerberoasting** — an attack where service tickets are cracked offline to recover service account passwords. This STIG enforces AES-only encryption.

## How we are fixing it
The value is a **bitmask** where each bit represents one encryption type. The required value `0x7FFFFFF8` (2147483640) enables AES128, AES256, and future encryption types only — disabling all DES and RC4 variants.

| Field | Value |
|---|---|
| Key  | `HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\Kerberos\Parameters` |
| Name | `SupportedEncryptionTypes` |
| Type | `REG_DWORD` |
| Data | `0x7FFFFFF8` (2147483640) |

Bitmask breakdown:
- `0x8` = AES128_HMAC_SHA1 ✅
- `0x10` = AES256_HMAC_SHA1 ✅
- `0x7FFFFFE0` = Future encryption types ✅
- DES and RC4 bits = unset ❌

**PowerShell (run as Administrator):**

```powershell
$path = "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\Kerberos\Parameters"

# Create the key path if it doesn't exist
if (-not (Test-Path $path)) {
    New-Item -Path $path -Force | Out-Null
}

# Set bitmask: AES128 + AES256 + future encryption types only
New-ItemProperty -Path $path -Name "SupportedEncryptionTypes" -Value 0x7FFFFFF8 -PropertyType DWORD -Force

# Local verify
Get-ItemProperty -Path $path -Name "SupportedEncryptionTypes"
```
