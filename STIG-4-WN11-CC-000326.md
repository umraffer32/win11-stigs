## What is the STIG ID?
**WN11-CC-000326** — PowerShell script block logging must be enabled on Windows 11.

When enabled, every PowerShell script block (including dynamically generated, encoded, or obfuscated code) gets logged to **Event ID 4104** in the `Microsoft-Windows-PowerShell/Operational` log. PowerShell deobfuscates the script before logging, so even `-EncodedCommand` Base64 payloads end up logged as plaintext. This is one of the most important logging controls for catching PowerShell-based malware.

## How I'm fixing it
Single registry DWORD. The policy path doesn't exist by default.

| Field | Value |
|---|---|
| Key  | `HKLM\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ScriptBlockLogging` |
| Name | `EnableScriptBlockLogging` |
| Type | `REG_DWORD` |
| Data | `1` |

**PowerShell (run as Administrator):**

```powershell
$path = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ScriptBlockLogging"

# Create key path if it doesn't exist
if (-not (Test-Path $path)) {
    New-Item -Path $path -Force | Out-Null
}

# Enable script block logging
New-ItemProperty -Path $path -Name "EnableScriptBlockLogging" -Value 1 -PropertyType DWORD -Force

# Local verify
Get-ItemProperty -Path $path -Name "EnableScriptBlockLogging"
```
