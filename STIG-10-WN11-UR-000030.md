# STIG #10 — WN11-UR-000030

## What is the STIG ID?
**WN11-UR-000030** — The 'Back up files and directories' user right must only be assigned to the Administrators group.

The `SeBackupPrivilege` right lets the holder **bypass NTFS permissions** to read any file. By default, Windows grants this right to both Administrators and Backup Operators. Removing Backup Operators reduces the privilege footprint — only Administrators retain the ability to read protected files via backup APIs.

## How we are fixing it
User Rights Assignments are stored in the local security policy database, not the registry. We use `secedit.exe` to export the policy, modify the relevant line, and re-import.

| Setting | Value |
|---|---|
| Right | `SeBackupPrivilege` |
| Required | `*S-1-5-32-544` (Administrators only) |
| SID `S-1-5-32-544` | Built-in Administrators group |

**PowerShell (run as Administrator):**

```powershell
# Step 1: Export current local security policy
secedit /export /cfg "$env:TEMP\secpol.cfg" /areas USER_RIGHTS

# Step 2: Replace the SeBackupPrivilege line to allow only Administrators
$cfg = Get-Content "$env:TEMP\secpol.cfg"
$cfg = $cfg -replace '^SeBackupPrivilege\s*=.*', 'SeBackupPrivilege = *S-1-5-32-544'
$cfg | Set-Content "$env:TEMP\secpol.cfg"

# Step 3: Re-import the modified policy
secedit /configure /db "$env:TEMP\secedit.sdb" /cfg "$env:TEMP\secpol.cfg" /areas USER_RIGHTS

# Step 4: Local verify
secedit /export /cfg "$env:TEMP\verify.cfg" /areas USER_RIGHTS
Select-String -Path "$env:TEMP\verify.cfg" -Pattern "^SeBackupPrivilege"
```

Expected verify output:
```
SeBackupPrivilege = *S-1-5-32-544
```
