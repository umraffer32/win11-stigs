## What is the STIG ID?
**WN11-SO-000215** — The system must be configured to meet the minimum session security requirement for NTLM SSP-based clients.

When Windows uses NTLM authentication (SMB, RPC, legacy services), this setting forces the client to demand strong cryptographic protection on the session: NTLMv2 session security and 128-bit encryption. Without these, an attacker performing MITM or relay attacks (e.g., via Responder) can negotiate a weaker session and recover credentials far more easily.

## How I'm fixing it
Single registry DWORD configured as a bitmask. The required value `0x20080000` enables both NTLMv2 session security and 128-bit encryption.

| Field | Value |
|---|---|
| Key  | `HKLM\SYSTEM\CurrentControlSet\Control\Lsa\MSV1_0` |
| Name | `NTLMMinClientSec` |
| Type | `REG_DWORD` |
| Data | `0x20080000` (537395200) |

Bitmask breakdown:
- `0x00080000` = Require NTLMv2 session security
- `0x20000000` = Require 128-bit encryption
- Combined: `0x20080000` ✅

**PowerShell (run as Administrator):**

```powershell
$path = "HKLM:\SYSTEM\CurrentControlSet\Control\Lsa\MSV1_0"

# Set bitmask: Require NTLMv2 session security + Require 128-bit encryption
New-ItemProperty -Path $path -Name "NTLMMinClientSec" -Value 0x20080000 -PropertyType DWORD -Force

# Local verify
Get-ItemProperty -Path $path -Name "NTLMMinClientSec"
```
