## What is the STIG ID?
**WN11-SO-000167** — Remote calls to the Security Account Manager (SAM) must be restricted to Administrators.

The SAM database stores local account info and password hashes. By default, low-privilege remote callers can use RPC (`SAMR`) to enumerate accounts — the same technique tools like BloodHound use during recon. Restricting SAM access to Administrators only is a major chokepoint against credential reconnaissance.

## How I'm fixing it
Unlike most STIGs, the value here is not a number — it's an **SDDL string** (Security Descriptor Definition Language) that encodes "only Administrators can make remote SAM calls."

| Field | Value |
|---|---|
| Key  | `HKLM\SYSTEM\CurrentControlSet\Control\Lsa` |
| Name | `RestrictRemoteSAM` |
| Type | `REG_SZ` |
| Data | `O:BAG:BAD:(A;;RC;;;BA)` |

SDDL breakdown:
- `O:BA` = Owner: Built-in Administrators
- `G:BA` = Group: Built-in Administrators
- `D:(A;;RC;;;BA)` = DACL: **A**llow **R**ead **C**ontrol → **B**uilt-in **A**dministrators

**PowerShell (run as Administrator):**

```powershell
$path = "HKLM:\SYSTEM\CurrentControlSet\Control\Lsa"

# Set the SDDL string restricting remote SAM access to Administrators
New-ItemProperty -Path $path -Name "RestrictRemoteSAM" -Value "O:BAG:BAD:(A;;RC;;;BA)" -PropertyType String -Force

# Local verify
Get-ItemProperty -Path $path -Name "RestrictRemoteSAM"
```
