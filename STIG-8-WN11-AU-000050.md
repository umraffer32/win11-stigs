## What is the STIG ID?
**WN11-AU-000050** — The system must be configured to audit Detailed Tracking → Process Creation successes.

This is what generates **Event ID 4688** for every process creation on the system. Combined with `WN11-CC-000066` (command line in process creation events), it produces full process telemetry — what ran and what arguments were used. Foundation of host-based detection.

## How I'm fixing it

### Prerequisite: WN11-SO-000030 (force subcategory audit policy)
By default, legacy audit categories override subcategory settings, which would silently ignore our `auditpol` change. Setting `SCENoApplyLegacyAuditPolicy = 1` forces Windows to honor the granular subcategories.

| Field | Value |
|---|---|
| Key  | `HKLM\SYSTEM\CurrentControlSet\Control\Lsa` |
| Name | `SCENoApplyLegacyAuditPolicy` |
| Type | `REG_DWORD` |
| Data | `1` |

```powershell
$path = "HKLM:\SYSTEM\CurrentControlSet\Control\Lsa"
New-ItemProperty -Path $path -Name "SCENoApplyLegacyAuditPolicy" -Value 1 -PropertyType DWORD -Force
Get-ItemProperty -Path $path -Name "SCENoApplyLegacyAuditPolicy"
```
<img width="1470" height="426" alt="image" src="https://github.com/user-attachments/assets/bdb97144-79b6-4b4e-9dd9-a72ee8d6f25f" />

### Main fix: Enable Process Creation auditing
This STIG uses `auditpol.exe` rather than a registry value. Run as Administrator:

```powershell
# Set Detailed Tracking → Process Creation to audit Successes
auditpol /set /subcategory:"Process Creation" /success:enable

# Local verify
auditpol /get /subcategory:"Process Creation"
```

Expected verify output:
```
Process Creation                        Success
```
